- **프로젝트명**: 사내 지식 AI 검색 시스템
- **작성일**: 2026-05-08
- **버전**: 1.0
- **작성자**: AI Assistant

## 1. 전체 레이아웃 구조

### 1.1 기본 레이아웃 (Layout Component)
모든 주요 화면에서 사용하는 기본 레이아웃 구조입니다.

**구성 요소:**
- **사이드바** (Sidebar): 좌측 고정, 네비게이션 메뉴
- **헤더** (Header): 상단 고정, 사용자 정보 및 로그아웃
- **메인 콘텐츠** (Main): 중앙 영역, 각 페이지별 콘텐츠 표시

**반응형 동작:**
- **데스크톱 (1024px 이상)**:
  - 사이드바가 항상 표시됨 (고정)
  - 헤더에 햄버거 메뉴 버튼 숨김
  - 메인 콘텐츠 최대 너비 제한 (페이지별 상이)

- **태블릿 (768px ~ 1023px)**:
  - 사이드바가 오버레이 형태로 표시
  - 햄버거 메뉴 버튼 표시
  - 메인 콘텐츠가 전체 너비 사용

- **모바일 (767px 이하)**:
  - 사이드바가 전체 화면 오버레이로 표시
  - 햄버거 메뉴 버튼 표시
  - 모든 텍스트와 버튼 크기 축소
  - 패딩과 간격 축소

## 2. 공통 컴포넌트

### 2.1 헤더 (Header)

**구성 요소:**
- **햄버거 메뉴 버튼**: 모바일/태블릿에서만 표시
- **앱 타이틀**: "사내 지식 AI 검색 시스템"
- **사용자 정보**: 이름, 역할 (일반 사용자/관리자)
- **로그아웃 버튼**: 로그아웃 기능

**반응형 동작:**
- **데스크톱**: 
  - 타이틀 text-xl, 모든 정보 표시
  - 사용자 정보 박스 표시
  
- **태블릿**:
  - 햄버거 메뉴 버튼 표시
  - 사용자 정보 박스 표시
  
- **모바일**:
  - 타이틀 text-base로 축소
  - 사용자 정보 박스 숨김
  - 로그아웃 버튼에서 텍스트 숨김 (아이콘만 표시)

**인터랙션:**
- 햄버거 메뉴 클릭 → 사이드바 토글
- 로그아웃 버튼 클릭 → 로그인 페이지로 이동

### 2.2 사이드바 (Sidebar)

**구성 요소:**
- **닫기 버튼**: 모바일/태블릿에서만 상단에 표시
- **사용자 메뉴 섹션**:
  - 통합 검색 (/)
  - AI 질문 (/ai-search)
  
- **관리자 메뉴 섹션** (관리자 권한시만 표시):
  - 관리자 대시보드 (/admin/dashboard)
  - 데이터 소스 설정 (/admin/datasource-settings)
  - 파일 현황 분석 (/admin/file-analytics)
  - 인덱싱 관리 (/admin/indexing)
  - 제외 정책 관리 (/admin/exclusion-policy)
  - 사용자 관리 (/admin/users)
  - 작업 로그 (/admin/logs)
  - 분석 실패 파일 (/admin/failed-files)
  - RAG/LLM 설정 (/admin/rag-settings)
  - 중복 문서 탐지 (/admin/duplicate-detection)

**반응형 동작:**
- **데스크톱 (1024px 이상)**:
  - 왼쪽에 항상 표시 (너비 256px)
  - 닫기 버튼 숨김
  
- **태블릿/모바일 (1023px 이하)**:
  - 기본적으로 숨김 (transform: translateX(-100%))
  - 햄버거 메뉴 클릭시 왼쪽에서 슬라이드인
  - 배경 오버레이 표시 (검은색 50% 투명도)
  - 닫기 버튼 표시
  - 메뉴 항목 클릭시 자동으로 닫힘

**인터랙션:**
- 메뉴 항목 클릭 → 해당 페이지로 이동 (모바일에서 사이드바 자동 닫힘)
- 현재 활성 경로는 파란색 배경으로 하이라이트
- 배경 오버레이 클릭 → 사이드바 닫힘

## 3. 사용자 화면

### 3.1 통합 검색 (SearchPage)
**경로**: `/`

#### 화면 구성

**1. 페이지 제목 영역**
- 타이틀: "통합 검색"
- 서브 타이틀: "사내 ownCloud의 문서와 소스 파일을 검색하세요"

**2. 검색 카드**
- **검색 입력 필드**: 
  - 플레이스홀더: "파일명, 경로, 본문, 코드 내용 검색..."
  - 실시간 입력 가능
- **검색 버튼**: "검색" 텍스트, 파란색 Primary 버튼
- **고급 필터 토글 버튼**: 
  - 텍스트: "고급 필터 표시/숨기기"
  - 클릭시 필터 영역 확장/축소

**3. 고급 필터 영역** (확장시 표시)
- **데이터 소스 선택**: 전체 / 사내 문서함 / 고객사 A WebDAV / 테스트 WebDAV
- **확장자 입력**: 예) pdf, docx, java
- **문서 유형 선택**: 전체 / 문서 / 소스 코드 / 이미지 / 압축 파일
- **파일 경로 입력**: 예) /documents/2024
- **수정일 (시작)**: 날짜 선택기
- **수정일 (종료)**: 날짜 선택기
- **파일 크기 선택**: 전체 / 1MB 이하 / 1MB-10MB / 10MB-100MB / 100MB 이상

**4. 검색 결과 영역**
- **결과 카운트**: "총 X개의 검색 결과"
- **결과 카드 리스트**: 각 결과별 카드 형태로 표시

**검색 결과 카드 구성:**
- 파일 아이콘
- 데이터 소스 뱃지 (파란색)
- 파일명 (클릭 가능, 파일 상세 페이지로 이동)
- 확장자 뱃지 (파일 타입별 색상)
- 관련도 점수 (%)
- 파일 경로
- 수정일
- 검색 위치 뱃지 (본문/파일명/코드)
- 미리보기 텍스트 (회색 배경 박스)
- 액션 버튼:
  - 상세 보기
  - 경로 복사
  - 원본 저장소

#### 반응형 동작

**데스크톱:**
- 최대 너비: 1152px (max-w-6xl)
- 고급 필터 3열 그리드 (lg:grid-cols-3)
- 검색 결과 카드 가로 레이아웃 (좌측: 정보, 우측: 버튼)

**태블릿:**
- 고급 필터 2열 그리드 (md:grid-cols-2)
- 검색 결과 카드 가로 레이아웃 유지

**모바일:**
- 고급 필터 1열 그리드
- 검색 결과 카드 세로 레이아웃으로 전환
- 버튼이 가로로 나열 (flex-row)
- 파일명과 경로 줄바꿈 허용 (break-all)
- 텍스트 크기 축소 (text-xs)
- 버튼 텍스트 축소 ("경로 복사" → "복사")

#### 상태 관리
- `searchQuery`: 검색어 입력값
- `showFilters`: 고급 필터 표시 여부

#### 데이터 예시
```javascript
{
  id: 1,
  fileName: 'product_spec_v2.pdf',
  filePath: '/documents/products/2024/product_spec_v2.pdf',
  extension: 'pdf',
  modifiedDate: '2024-12-15',
  relevanceScore: 95,
  searchLocation: '본문',
  dataSource: '사내 문서함',
  preview: '제품 사양서 업데이트 내용...'
}
```

## 4. 관리자 화면

### 4.1 관리자 대시보드 (DashboardPage)
**경로**: `/admin/dashboard`

#### 화면 구성

**1. 페이지 제목 영역**
- 타이틀: "관리자 대시보드"
- 서브 타이틀: "시스템 전체 현황을 확인하세요"

**2. 통계 카드 영역**
5개의 통계 카드를 가로로 배치:
- **전체 파일**: 파일 아이콘, 숫자 표시
- **분석 완료**: 체크 아이콘(녹색), 숫자 표시
- **분석 실패**: X 아이콘(빨간색), 숫자 표시
- **대기 중**: 시계 아이콘(노란색), 숫자 표시
- **Vector DB**: 데이터베이스 아이콘(보라색), 청크 수 표시

**3. 인덱싱 상태 카드**
- **제목**: "최근 인덱싱 상태"
- **상태 뱃지**: 진행 중 / 완료 / 실패
- **진행률 바**: 퍼센트 표시 (0-100%)
- **통계 박스 3개**:
  - 처리 완료: 숫자
  - 실패: 숫자 (빨간색)
  - 남은 수: 숫자
- **현재 분석 중인 파일**: 파란색 배경 박스에 파일 경로 표시
- **액션 버튼 4개**:
  - 전체 분석 시작 (파란색)
  - 변경분 분석 (외곽선)
  - 분석 중지 (빨간색)
  - 데이터 소스 테스트 (회색)

**4. 관리자 작업 로그 카드**
- **제목**: "최근 관리자 작업 로그"
- **로그 항목 리스트**: 
  - 시간
  - 사용자명
  - 작업 내용
  - 상태 뱃지 (성공/실패)

#### 반응형 동작

**데스크톱:**
- 최대 너비: 1280px (max-w-7xl)
- 통계 카드: 5열 그리드 (lg:grid-cols-5)
- 인덱싱 통계: 3열 그리드 (md:grid-cols-3)
- 액션 버튼: 가로 나열 (flex)
- 로그 항목: 가로 레이아웃

**태블릿:**
- 통계 카드: 2열 그리드 (md:grid-cols-2)
- 액션 버튼: 가로 나열 유지
- 로그 항목: 가로 레이아웃 유지

**모바일:**
- 통계 카드: 1열 그리드
- 인덱싱 통계: 1열 그리드
- 액션 버튼: 2x2 그리드 레이아웃으로 변경
- 버튼 텍스트 축소 ("전체 분석 시작" → "전체 분석")
- 로그 항목: 세로 레이아웃으로 전환
- 텍스트 크기 축소 (text-xs)

#### 데이터 예시
```javascript
// 통계
stats: {
  totalFiles: 12580,
  analyzedFiles: 11245,
  failedFiles: 235,
  pendingFiles: 1100,
  vectorDBChunks: 45680
}

// 인덱싱 상태
recentIndexing: {
  status: '진행 중',
  progress: 67,
  currentFile: '/documents/2024/reports/annual_report.pdf',
  processed: 7540,
  failed: 15,
  remaining: 3645
}

// 로그
{
  id: 1,
  time: '2024-12-15 14:30',
  user: '홍길동',
  action: '인덱싱 시작',
  status: 'success'
}
```

## 9. 버튼 스타일

### 9.1 버튼 Variants
- **Primary**: 파란색 배경, 흰색 텍스트
- **Secondary**: 회색 배경, 검은색 텍스트
- **Outline**: 투명 배경, 파란색 테두리
- **Ghost**: 투명 배경, 회색 텍스트 (hover시 배경)
- **Danger**: 빨간색 배경, 흰색 텍스트

### 9.2 버튼 크기
- **sm**: 작은 크기 (padding 축소)
- **md**: 기본 크기
- **lg**: 큰 크기

### 9.3 모바일 최적화
- 모바일에서 버튼 텍스트 축소 또는 숨김
- 아이콘만 표시하거나 간략화된 텍스트 사용
- 터치하기 쉬운 최소 크기 유지 (44px x 44px)

```
POST /api/admin/indexing/stop              → 인덱싱 중지
GET /api/admin/logs                        → 작업 로그
GET /api/admin/users                       → 사용자 목록
POST /api/auth/login                       → 로그인
POST /api/auth/register                    → 회원가입
POST /api/auth/logout                      → 로그아웃
```
