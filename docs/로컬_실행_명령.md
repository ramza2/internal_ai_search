# 로컬 실행 명령 (Quick Reference)

> **목적:** API 서버 · 백그라운드 worker · 프론트엔드를 띄울 때 쓰는 **명령만** 모아 둔 문서입니다.  
> 최초 설치(venv, `pip install`, DB 마이그레이션, Ollama, `.env`)는 [`backend/README.md`](../backend/README.md) · [`frontend/README.md`](../frontend/README.md) 를 참고하세요.

---

## 한눈에 보기 (터미널 3개)

| # | 역할 | 작업 디렉터리 | 명령 요약 |
|---|------|---------------|-----------|
| 1 | **API** | `backend/` | `uvicorn app.main:app --reload …` |
| 2 | **Worker** (백그라운드 Job) | `backend/` | `python -m app.worker_main` |
| 3 | **Frontend** | `frontend/` | `npm run dev` (개발) 또는 `build` + `preview` |

- **동기 API만** 쓸 때(예: curl로 `process-pending-documents` 직접 호출): Worker 없이 **1번만**으로도 가능합니다.
- **Pipeline / admin Job enqueue** (`PENDING` → 처리): **1번 + 2번** 이 필요합니다.
- UI로 검색·관리: **1번 + 3번** (개발 시 Vite 프록시 → `localhost:8000`).

---

## 0) 사전 준비 (최초 1회)

```bash
# backend — backend/README.md §1
cd backend
python -m venv .venv
# Windows: .venv\Scripts\Activate.ps1
# Linux/macOS: source .venv/bin/activate
pip install -r requirements.txt
# backend/.env 작성 (DB, JWT, Ollama 등)

# frontend — frontend/README.md
cd frontend
npm install
```

PostgreSQL·마이그레이션·Ollama(`bge-m3`, `gemma3` 등)는 백엔드 README를 따릅니다.

## 1) Backend API (uvicorn)

**작업 디렉터리:** `backend/` (가상환경 활성화 후)

### Windows (PowerShell)

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### Linux / macOS / WSL

```bash
cd backend
source .venv/bin/activate
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

- **확인:** `http://localhost:8000/health`
- **API 문서:** `http://localhost:8000/docs` (FastAPI Swagger)

---

## 2) Background worker

Worker는 **별도 프로세스**입니다. API(uvicorn)와 **같은 `backend/`·같은 venv**를 쓰되, **터미널을 분리**하세요.  
별도 “빌드” 단계는 없습니다 (`pip install -r requirements.txt` 로 충분).

### Windows (PowerShell)

```powershell
cd backend
.\.venv\Scripts\Activate.ps1
python -m app.worker_main
```

### Linux / macOS / WSL

```bash
cd backend
source .venv/bin/activate
python -m app.worker_main
```

- `WORKER_ENABLED=false`(기본)여도 **CLI로 실행하면** 큐의 `PENDING` Job을 처리합니다.
- 처리 대상 예: `WEBDAV_SYNC_TREE`, `PROCESS_PENDING_TEXT`, `PROCESS_PENDING_DOCUMENTS`, `CHUNK_COMPLETED_TEXT`, `EMBED_PENDING_CHUNKS`, `PIPELINE` 등.
- **중지:** 해당 터미널에서 `Ctrl+C`.

상세: [`docs/07_아키텍처/백그라운드작업_워커구조.md`](./07_아키텍처/백그라운드작업_워커구조.md), `backend/README.md` — DB polling worker.

## 3) Frontend

**작업 디렉터리:** `frontend/`

### 개발 서버 (일반)

```bash
cd frontend
npm run dev
```

- **URL:** `http://localhost:5173`
- `VITE_API_BASE_URL` 을 **비우면** `/api` → Vite 프록시 → `http://localhost:8000` (`vite.config.ts`).

### 프로덕션 빌드 + 로컬 미리보기

```bash
cd frontend
npm run build
npm run preview
```

- `build`: `tsc -b && vite build` → 산출물 `frontend/dist/`
- `preview`: 빌드 결과를 로컬에서 서빙 (기본 포트는 Vite preview 설정 따름, 보통 `4173` 근처 — 터미널 출력 확인).

운영 배포 시에는 `dist/` 를 정적 호스팅하고, API는 별도 URL이면 `VITE_API_BASE_URL` 에 백엔드 절대 URL을 넣습니다.

## 4) 자주 쓰는 조합

### UI 개발 + 동기 API만

1. `uvicorn` (터미널 1)  
2. `npm run dev` (터미널 2)  
3. Worker **불필요** — 데이터 소스 화면에서 동기 `sync-tree` / `process-pending-documents` 등 직접 호출 시

### UI + 백그라운드 Job (Pipeline 모달 “백그라운드 실행”)

1. `uvicorn` (터미널 1)  
2. `python -m app.worker_main` (터미널 2)  
3. `npm run dev` (터미널 3)

### API·Worker만 (curl / E2E)

1. `uvicorn`  
2. `python -m app.worker_main` (백그라운드 enqueue 검증 시)  
3. Frontend **생략 가능**
