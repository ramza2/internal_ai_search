# 서버 배포 가이드

> **대상 서버:** Ubuntu (192.168.10.220)  
> **Ollama (GPU):** 개발 PC (192.168.10.154)  
> **배포 방식:** Docker Compose (`docker-compose.prod.yml`)

---

## 네트워크 구성

```
[사용자 브라우저]
    │
    ├── :5173  ──→  frontend (Vite)  ──proxy──→  backend:8000 (컨테이너 내부)
    │
[192.168.10.220 서버]
    ├── :8100  ──→  backend (FastAPI)   ← API 직접 호출 시
    ├── :5434  ──→  db (PostgreSQL)     ← 127.0.0.1만 바인딩
    │
    └──── Ollama 요청 ──→  192.168.10.154:11434 (개발 PC GPU)
```

### 포트 요약

| 서비스 | 호스트 포트 | 컨테이너 포트 | 비고 |
|--------|-------------|---------------|------|
| frontend | 5173 | 5173 | 브라우저 접속용 |
| backend | 8100 | 8000 | API 직접 호출 시 |
| db | 5434 (localhost만) | 5432 | 외부 접근 차단 |
| Ollama | — | — | 192.168.10.154:11434 (외부 서버) |

## 0단계. 사전 준비

### 0-1. 서버 요구사항

- Ubuntu 20.04+
- Docker Engine 24+
- Docker Compose v2+ (`docker compose` 서브커맨드)
- Git
- 인터넷 접속 (Docker 이미지 pull, GitHub clone)

### 0-2. 개발 PC Ollama 외부 접근 허용 (192.168.10.154)

Ollama는 기본적으로 `127.0.0.1:11434`만 수신합니다. 서버에서 접근하려면 외부 바인딩이 필요합니다.

**Windows (PowerShell, 관리자):**

```powershell
# 시스템 환경변수 추가 (1회)
[System.Environment]::SetEnvironmentVariable("OLLAMA_HOST", "0.0.0.0:11434", "User")

# Ollama 재시작: 트레이 아이콘 → Quit → 다시 실행
```

**Linux/macOS:**

```bash
# ~/.bashrc 또는 systemd service에 추가
export OLLAMA_HOST=0.0.0.0:11434
# Ollama 재시작
```

**확인 (개발 PC에서):**

```bash
curl http://192.168.10.154:11434/api/tags
```

모델 목록이 JSON으로 나오면 정상입니다.

### 0-3. 필요 Ollama 모델 확인

개발 PC에 아래 모델이 pull 되어 있어야 합니다:

```bash
ollama list
# 필요: bge-m3 (임베딩), gemma3 (RAG/AI 질문)

# 없으면:
ollama pull bge-m3
ollama pull gemma3
```

### 0-4. 방화벽 확인

| 출발지 | 목적지 | 포트 | 용도 |
|--------|--------|------|------|
| 사용자 PC | 192.168.10.220 | 5173 | 웹 UI |
| 사용자 PC | 192.168.10.220 | 8100 | API 직접 호출 (선택) |
| 192.168.10.220 | 192.168.10.154 | 11434 | Ollama |

## 2단계. 소스 클론

```bash

```

> **인증:** GitHub에서 Password 인증은 지원하지 않습니다.  
> **Personal Access Token (PAT)** 을 발급하여 비밀번호 대신 입력하세요.  
> GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token

## 3단계. 환경 파일 작성

```bash
cd /opt/internal_ai_search
cp backend/.env.prod.example backend/.env.prod
nano backend/.env.prod
```

### 반드시 변경할 항목 (CHANGE_ME)

| 변수 | 설명 | 생성 방법 |
|------|------|-----------|
| `DB_PASSWORD` | PostgreSQL 비밀번호 | `openssl rand -hex 16` |
| `JWT_SECRET_KEY` | JWT 서명키 | `openssl rand -hex 32` |
| `DATA_SOURCE_SECRET_KEY` | 데이터소스 암호화키 | `openssl rand -hex 32` |
| `INITIAL_ADMIN_PASSWORD` | 초기 관리자 비밀번호 | 직접 설정 (최초 로그인 시 변경됨) |

### 확인할 항목

| 변수 | 기본값 | 확인 |
|------|--------|------|
| `OLLAMA_BASE_URL` | `http://192.168.10.154:11434` | 개발 PC IP 맞는지 |
| `OLLAMA_MODEL` | `gemma3` | 개발 PC에 pull 되어 있는지 |
| `EMBEDDING_MODEL` | `bge-m3` | 개발 PC에 pull 되어 있는지 |
| `WORKER_ENABLED` | `true` | 백그라운드 Job 처리 필요 시 |

```bash
# 키 생성 도우미
openssl rand -hex 16   # DB_PASSWORD 용
openssl rand -hex 32   # JWT_SECRET_KEY 용
openssl rand -hex 32   # DATA_SOURCE_SECRET_KEY 용
```

## 4단계. 빌드 및 실행

```bash
cd /opt/internal_ai_search

# Worker 포함 전체 스택 빌드 + 실행
docker compose \
  --env-file backend/.env.prod \
  -f docker-compose.prod.yml \
  --profile worker \
  up -d --build
```

최초 빌드는 pip install / npm ci 때문에 **3~5분** 걸릴 수 있습니다.

## 운영 명령 모음

### 별칭 설정 (선택)

```bash
# ~/.bashrc에 추가하면 편리합니다
alias dc="docker compose --env-file backend/.env.prod -f docker-compose.prod.yml"
```

### 상태 확인

```bash
dc ps                           # 컨테이너 상태
dc logs -f backend              # 백엔드 로그 (실시간)
dc logs -f backend-worker       # Worker 로그
dc logs -f frontend             # 프론트 로그
dc logs --tail 50               # 전체 최근 50줄
```

### 재시작

```bash
dc restart backend              # 백엔드만 재시작
dc restart backend-worker       # Worker만 재시작
dc restart frontend             # 프론트만 재시작
```

### 소스 업데이트 후 재배포

```bash
cd /opt/internal_ai_search
git pull

# 변경된 서비스만 재빌드 + 재시작
dc up -d --build

# 또는 특정 서비스만
dc up -d --build backend frontend
```

### 중지

```bash
dc down                         # 전체 중지 (DB 데이터 유지)
dc down -v                      # 전체 중지 + DB 데이터 삭제 (초기화)
```

## 문제 해결

### Ollama 연결 실패

```
httpx.ConnectError: ... 192.168.10.154:11434
```

1. 개발 PC에서 Ollama가 실행 중인지 확인
2. `OLLAMA_HOST=0.0.0.0:11434` 환경변수 설정 후 Ollama 재시작
3. 방화벽에서 11434 포트가 열려 있는지 확인
4. 서버에서 `curl http://192.168.10.154:11434/api/tags` 확인

### DB 연결 실패

```bash
# DB 컨테이너 상태 확인
dc logs db --tail 20

# DB 볼륨 초기화 (데이터 전부 삭제)
dc down -v
dc up -d --build
```

### frontend에서 API 호출 실패

frontend는 컨테이너 내부에서 `backend:8000`으로 프록시합니다.
backend 컨테이너가 정상 기동되었는지 확인하세요:

```bash
dc logs backend --tail 20
```

### 포트 충돌

```bash
# 어떤 프로세스가 포트를 쓰고 있는지 확인
sudo lsof -i :8100
sudo lsof -i :5173
```

`docker-compose.prod.yml`의 `ports:` 값을 변경하여 다른 포트로 조정할 수 있습니다.

## 보안 참고

- `backend/.env.prod`는 **Git에 커밋하지 마세요** (`.gitignore`에 포함됨)
- DB 포트(5434)는 `127.0.0.1`만 바인딩 — 서버 외부에서 직접 접근 불가
- 초기 관리자 비밀번호는 최초 로그인 시 반드시 변경
- JWT_SECRET_KEY, DATA_SOURCE_SECRET_KEY는 충분히 긴 랜덤 문자열 사용
